CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. Example: (h1 style="color:red;">Prework Study Guide/h1 dont forget to add the "<>")
  • This internal style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code. Example: ( style h1 { color: red; } /style /head) The preceding CSS states that all h1 elements on the webpage will be red. dont forget the "<>"
  • External css style sheet is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier. When using an external style sheet, we link the file using a element placed in the HTML's , as shown in the following code: (head link rel="stylesheet" href="./assets/style.css" /> head)
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

JavaScript

  • Using JavaScript, you can power your websites with effects, interactivity, data communication, and much more.
  • In coding parlance, a variable is simply a named container that allows us to store data in our code. One of its core features is variables, which store data types as values.
  • In JavaScript, to declare a variable, we use the var keyword. for example: var shape = "circle"; In this example, "shape" is the name of our container, or variable, and the data it stores is "circle".
  • Finally, we use an assignment operator "=" to assign the specific data that we want stored in our variable.
  • the string is contained within quotation marks, ("). All strings are surrounded by quotation marks.
  • We can use the console.log() method to output a message to the web console by adding an argument. In this case, console.log(topics) will output the value of topics to the web console because we've added it as an argument inside the parentheses.
  • console.log outputs the value of a variable.
  • To understand how to use JavaScript, we first need to understand the concept of control flow. Control flowis the order in which a computer executes code in a script.
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.To create an array, we declare a variable in the same way that we declared a variable with only one data item. instead of assigning the variable to a single value, we assign the variable to a group of values. For example: var shapes = ["triangle", "square", "pentagon", "circle"];
  • the location of the data item in the array. This location is identified with a unique number called an index.
  • A "for" loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. there are three important statements: starting point for our loop(var x=0). The second statement is the condition(x < shapes.length). We use the length built-in array property on our shapes array to express that condition. The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code x++ is a shortcut way of writing x+1. Now every time the code loops, x will increase by 1, as shown in the following line of code (x++). All together a "for" loop looks like the following: for(var x = 0; x < shapes.length; x++)
  • Commenting out code makes it inactive without deleting it, just in case you need it later. To comment out multiple lines of JavaScript code, put /* before and */ after the block of code
